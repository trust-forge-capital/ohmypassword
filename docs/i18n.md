# Internationalization (i18n) Design

## Overview

ohmypassword supports multiple languages to accommodate users worldwide. The i18n system is designed to be lightweight and extensible.

## Supported Languages

| Code | Language | Status |
|------|----------|--------|
| en | English | Default |
| zh | Chinese (Simplified) | Supported |
| zh-TW | Chinese (Traditional) | Supported |
| ja | Japanese | Supported |
| ko | Korean | Supported |
| es | Spanish | Supported |
| fr | French | Supported |

## Language Selection Priority

1. CLI flag: `-L` or `--lang`
2. System locale detection
3. Default: English

## Implementation

### Message Structure

```go
type MessageMap map[string]string
```

### Message Keys

All user-facing strings are keyed:

```go
// Command descriptions
"root_use"
"root_long"
"generate_use"
"generate_long"

// Flags
"flag_length"
"flag_charset"
"flag_strategy"
"flag_count"
"flag_validate"
"flag_quiet"
"flag_lang"
"flag_output"

// Output
"output_entropy"
"output_strength"
"output_crack_time"

// Strength levels
"strength_very_weak"
"strength_weak"
"strength_reasonable"
"strength_strong"
"strength_very_strong"

// Errors
"error_invalid_length"
"error_invalid_count"
"error_invalid_strategy"
"error_invalid_charset"
```

### Language Files

Each language is in its own file:
```
internal/i18n/messages/
├── messages.go   # English (base)
├── zh.go         # Simplified Chinese
├── zh-TW.go      # Traditional Chinese
├── ja.go         # Japanese
├── ko.go         # Korean
├── es.go         # Spanish
└── fr.go         # French
```

### Usage Example

```go
import "github.com/trust-forge-capital/ohmypassword/internal/i18n"

// Set language
i18n.SetLanguage("zh")

// Get translated string
fmt.Println(i18n.T("generate_use"))  // 输出: 生成新密码

// Format with arguments
fmt.Println(i18n.TFormat("password_length", 16))  // 输出: 密码长度: 16
```

## Adding New Languages

1. Create new file in `internal/i18n/messages/`
2. Copy English messages as template
3. Translate all strings
4. Register in `locale.go` init function
5. Update help text in CLI commands

## Unicode Considerations

- All translations use UTF-8 encoding
- Terminal must support UTF-8 for proper display
- No hardcoded byte values
- Bidirectional text not supported (future enhancement)
